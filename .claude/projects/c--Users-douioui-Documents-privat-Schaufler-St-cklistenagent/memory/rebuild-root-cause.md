The audit framing "system is Vision-First for born-digital PDFs, that's unpatchable" is **wrong for the current code**. `structure_normalizer.parse_file` already routes text-layer PDFs to `parse_pdf_text` and reserves Vision for true scans. BUT `parse_pdf_text` still hands coordinate-banded text to an LLM for row/column segmentation â€” so the stochastic step is LLM table-segmentation, not Vision. The deterministic coordinate machinery (`_build_spatial_rows`, `_build_column_profile`, `_cross_check_with_text_layer` in `src/ingestion/pdf_parser.py`) already exists but is wired as a *verifier*, not as the authoritative extractor.

**The real bug behind T-007 (N parts under one Pos-Nr â†’ 1 row) and the nameless-rows loss:** the entire zero-data-loss chain is keyed on the **position identifier as row identity** â€” `_collect_position_values` dedupes by position, the reconciler `master_set` is position-keyed, the B3 coverage guard and the export set-guard compare position sets. Every checkpoint counts distinct positions, never distinct rows. So rows sharing a position (or having none) are invisible to the guard by construction. A Text-First rebuild that keeps a position/count anchor does NOT fix this â€” row identity must become a deterministic spatial band, not the position value.

**Recommendation given:** not full greenfield (throws away working no-false-green scoring + V191 export), not endless patching (guard is structurally blind). Targeted spine rebuild: invert ingestion to deterministic coordinate reconstruction + re-key row identity from position to spatial band. ~2.5â€“4 weeks realistic.
